# Code Conventions

## JavaScript
- Use ES6+ syntax
- Follow camelCase for variable and function names
- Use `const` for variables that don't change, `let` for those that do
- Avoid global variables when possible
- Use meaningful variable and function names
- Write JSDoc-style comments for functions

## CSS
- Use BEM (Block Element Modifier) methodology for class naming
- Keep selectors as specific as needed but not more
- Use CSS variables for colors and other theme values
- Maintain consistent indentation (2 spaces)
- Group related styles together

## HTML
- Use semantic HTML elements where appropriate
- Keep attributes on separate lines for readability when there are many
- Use lowercase for all HTML elements and attributes
- Validate HTML to ensure it's well-formed

## General Principles
- Keep files modular and focused on single responsibilities
- Write code that is easy to read and understand
- Prefer clarity over cleverness in code structure
